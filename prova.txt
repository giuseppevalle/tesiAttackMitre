<p>Credential dumping is the process of obtaining account login and password information from the operating system and software. Credentials can be used to perform <a href="/wiki/Lateral_Movement" title="Lateral Movement">Lateral Movement</a> and access restricted information.
</p><p>Tools may dump credentials in many different ways: extracting credential hashes for offline cracking, extracting plaintext passwords, and extracting Kerberos tickets, among others. Examples of credential dumpers include pwdump7, <a href="/wiki/Software/S0005" title="Software/S0005">Windows Credential Editor</a>, <a href="/wiki/Software/S0002" title="Software/S0002">Mimikatz</a>, and <a href="/wiki/Software/S0008" title="Software/S0008">gsecdump</a>. These tools are in use by both professional security testers and adversaries.
</p><p>Plaintext passwords can be obtained using tools such as <a href="/wiki/Software/S0002" title="Software/S0002">Mimikatz</a> to extract passwords stored by the Local Security Authority (LSA). If smart cards are used to authenticate to a domain using a personal identification number (PIN), then that PIN is also cached as a result and may be dumped.<span id="scite-ref-9cab363c426b80c420cc0b022ff40fec-1-a" class="scite-citeref-number" data-reference="Github Mimikatz Module sekurlsa"><a href="#scite-9cab363c426b80c420cc0b022ff40fec">1</a></span>
</p><p>DCSync is a variation on credential dumping which can be used to acquire sensitive information from a domain controller. The action works by simulating a domain controller replication process from a remote domain controller, which may contain various pieces of information included in Active Directory such as passwords, historical hashes, and current hashes of potentially useful accounts, such as the KRBTGT account NTLM hash. Any members of the Administrators, Domain Admins, Enterprise Admin groups or computer accounts on the domain controller are able to run DCSync to pull password data.<span id="scite-ref-254cdc45583d463b3ba4c69880e905ae-2-a" class="scite-citeref-number" data-reference="ADSecurity Mimikatz DCSync"><a href="#scite-254cdc45583d463b3ba4c69880e905ae">2</a></span> The hashes can then in turn be used to create a Golden Ticket for use in <a href="/wiki/Technique/T1097" title="Technique/T1097">Pass the Ticket</a>.<span id="scite-ref-0c005d8c3134d5c5ab70a3bfe127d4a9-3-a" class="scite-citeref-number" data-reference="Harmj0y Mimikatz and DCSync"><a href="#scite-0c005d8c3134d5c5ab70a3bfe127d4a9">3</a></span> DCSync functionality has been included in the "lsadump" module in Mimikatz.<span id="scite-ref-37e18b924baf59c9bb150bcdb8a27760-4-a" class="scite-citeref-number" data-reference="GitHub Mimikatz lsadump Module"><a href="#scite-37e18b924baf59c9bb150bcdb8a27760">4</a></span>
</p><p>
</p>

<h2><span class="mw-headline" id="Mitigation">Mitigation</span></h2>
<p>Monitor/harden access to LSASS and SAM table with tools that allow process whitelisting. Limit credential overlap across systems to prevent lateral movement opportunities using <a href="/wiki/Technique/T1078" title="Technique/T1078">Valid Accounts</a> if passwords and hashes are obtained. Ensure that local administrator accounts have complex, unique passwords across all systems on the network. Do not put user or admin domain accounts in the local administrator groups across systems unless they are tightly controlled, as this is often equivalent to having a local administrator account with the same password on all systems. On Windows 8.1 and Windows Server 2012 R2, enable Protected Process Light for LSA.<span id="scite-ref-90040a6b65b80f4d794e3aa762c0bcec-37-a" class="scite-citeref-number" data-reference="Microsoft LSA"><a href="#scite-90040a6b65b80f4d794e3aa762c0bcec">37</a></span>
</p><p>Identify and block potentially malicious software that may be used to dump credentials by using whitelisting<span id="scite-ref-49226f502c38a4bd89296004a1c9a789-38-a" class="scite-citeref-number" data-reference="Beechey 2010"><a href="#scite-49226f502c38a4bd89296004a1c9a789">38</a></span> tools, like AppLocker,<span id="scite-ref-34da59ab7a40adec8a57dd53521c699e-39-a" class="scite-citeref-number" data-reference="Windows Commands JPCERT"><a href="#scite-34da59ab7a40adec8a57dd53521c699e">39</a></span><span id="scite-ref-7abde4164335b68f118c6476873ad933-40-a" class="scite-citeref-number" data-reference="NSA MS AppLocker"><a href="#scite-7abde4164335b68f118c6476873ad933">40</a></span> or Software Restriction Policies<span id="scite-ref-9e2a9d6c65e284c53af1d2456c4d25da-41-a" class="scite-citeref-number" data-reference="Corio 2008"><a href="#scite-9e2a9d6c65e284c53af1d2456c4d25da">41</a></span> where appropriate.<span id="scite-ref-f98899d7dc94d2fb51020ff4b9120a9c-42-a" class="scite-citeref-number" data-reference="TechNet Applocker vs SRP"><a href="#scite-f98899d7dc94d2fb51020ff4b9120a9c">42</a></span>
</p><p>With Windows 10, Microsoft implemented new protections called Credential Guard to protect the LSA secrets that can be used to obtain credentials through forms of credential dumping. It is not configured by default and has hardware and firmware system requirements.<span id="scite-ref-6244ed3130c1ea20f3629efdeec5cb06-43-a" class="scite-citeref-number" data-reference="TechNet Credential Guard"><a href="#scite-6244ed3130c1ea20f3629efdeec5cb06">43</a></span> It also does not protect against all forms of credential dumping.<span id="scite-ref-26bdfb5f997c753432359e6846a1a31a-44-a" class="scite-citeref-number" data-reference="GitHub SHB Credential Guard"><a href="#scite-26bdfb5f997c753432359e6846a1a31a">44</a></span>
</p>
<h2><span class="mw-headline" id="Detection">Detection</span></h2>
<p>Common credential dumpers such as <a href="/wiki/Software/S0002" title="Software/S0002">Mimikatz</a> access the LSA Subsystem Service (LSASS) process by opening the process, locating the LSA secrets key, and decrypting the sections in memory where credential details are stored. Credential dumpers may also use methods for reflective <a href="/wiki/Technique/T1055" title="Technique/T1055">DLL Injection</a> to reduce potential indicators of malicious activity.
</p><p>NTLM hash dumpers open the Security Accounts Manager (SAM) on the local file system (%SystemRoot%/system32/config/SAM) or create a dump of the Registry SAM key to access stored account password hashes. Some hash dumpers will open the local file system as a device and parse to the SAM table to avoid file access defenses. Others will make an in-memory copy of the SAM table before reading hashes. Detection of compromised <a href="/wiki/Technique/T1078" title="Technique/T1078">Valid Accounts</a> in-use by adversaries may help as well. 
</p><p>On Windows 8.1 and Windows Server 2012 R2, monitor Windows Logs for LSASS.exe creation to verify that LSASS started as a protected process.
</p><p>Monitor processes and command-line arguments for program execution that may be indicative of credential dumping. Remote access tools may contain built-in features or incorporate existing tools like <a href="/wiki/Software/S0002" title="Software/S0002">Mimikatz</a>. <a href="/wiki/Technique/T1086" title="Technique/T1086">PowerShell</a> scripts also exist that contain credential dumping functionality, such as PowerSploit's Invoke-Mimikatz module,<span id="scite-ref-84887288ab84f6c9ce88f535ea1b121f-45-a" class="scite-citeref-number" data-reference="Powersploit"><a href="#scite-84887288ab84f6c9ce88f535ea1b121f">45</a></span> which may require additional logging features to be configured in the operating system to collect necessary information for analysis.
</p>
